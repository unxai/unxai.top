Principles

Rules for growth

The hub should define the network, not compete with the products inside it.

One question per domain

New products should answer something distinct instead of overlapping with what already exists.

Small products first

It is better to ship a narrow, working site than to describe a giant platform with no edges.

Shared taste

Different forms are fine, as long as the network still feels like one system.

Future

Open for more domains later

Keep the hub short now, but leave room for more products under *.unxai.top.

New layers

More products can extend the network with tools, datasets, or public-facing systems.

Shared updates

A later version can aggregate releases, status, or changelog signals across the network.

Sharper identity

Once the structure is stable, the next gain is stronger visual rhythm, not more copy.
